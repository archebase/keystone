DEVELOPMENT DESIGN

Hilbert 接管 Keystone 数采员认证

凭据归 Hilbert；现场业务归 Keystone。

核心判断

本版本只切换数采员认证来源，不重构 Synapse，不引入 Hilbert logout，不改变 Keystone JWT 形态。

Keystone 在数采员登录时代理调用 Hilbert 验证账号密码。验证成功后，Keystone 仍使用本地 data_collectors 作为业务身份锚点，并签发现有 Keystone JWT。

新项目部署新环境，Keystone 必须配置 Hilbert。数采员登录没有本地密码兼容路径，Hilbert 不可用时直接返回认证服务不可用。

01

背景与问题

当前 Keystone 的数采员登录由 /auth/login 处理：管理员走环境变量，数采员走本地 data_collectors.operator_id + password_hash。

Hilbert 已经加入仓库并提供账号体系。你希望以后只在 Hilbert 上新建数采员，数采员在 Synapse 页面输入 Hilbert 用户名和密码，由 Keystone 与 Hilbert 交互完成验证。

核心矛盾：账号密码要单一来源，但 Keystone 不能失去本地采集员、工位、任务和统计的业务关系。
02

目标与非目标

本版本目标
  • Hilbert 启用后，数采员密码只由 Hilbert 验证。
  • Keystone 本地 collector 仍是业务身份锚点。
  • Admin 登录保持 Keystone 环境变量优先。
  • 登录成功后覆盖本地采集员姓名和工位冗余姓名。
  • 新增测试覆盖 Hilbert 成功、无绑定、认证失败、服务异常。
明确不做
  • 不改 Synapse 页面和接口调用。
  • 不保存 Hilbert sessionKey。
  • 不通知 Hilbert logout。
  • 不新增 auth_sessions 表。
  • 不修改 Keystone JWT claims。
  • 不重新生成 Swagger docs。
03

认证总览

Synapse 继续提交 operator_id/account + password 到 Keystone /auth/login。
Keystone 先尝试匹配本地 admin 环境变量账号。
若不是 admin，Keystone 调用 Hilbert nonce 与 login；未配置 Hilbert 视为服务配置错误。
Hilbert 验证成功后，Keystone 校验账号类型和状态。
Keystone 查找本地 data_collectors.operator_id = account.code 且 status = active。
Keystone 更新本地姓名与 last login，签发现有 JWT。
if admin_credentials_match:
  issue_keystone_admin_jwt()

if hilbert_base_url == "":
  fail_service_misconfigured()

account = hilbert_login(code, password)
require account.role == "external_user"
require account.externalUserType == "data_supplier"
require account.status == "enabled"

collector = find_active_collector(account.code)
sync_display_name(account.displayName)
issue_keystone_collector_jwt(collector)
04

Hilbert 协议

已验证真实 Hilbert 登录流程。先调用 nonce 接口获取 id 和 randomKey，再把明文密码做 SHA-256，使用 nonce material 的前 32 字节作为 AES-256 key，后 12 字节作为 GCM nonce，生成 cipherDigest。

本版本解析 Hilbert 返回的 sessionKey 以确认响应完整，但立即丢弃，不保存、不入 JWT、不写日志。

GET {base}/v1/console/nonce/generate

POST {base}/v1/console/account/login
{
  "code": "dc01",
  "nonceId": 67,
  "cipherDigest": "base64-aes-gcm"
}
05

准入与绑定规则

检查点 规则 失败结果
Hilbert 角色 role == external_user 401 invalid credentials
Hilbert 外部用户类型 externalUserType == data_supplier 401 invalid credentials
Hilbert 状态 status == enabled 401 invalid credentials
Keystone 绑定 必须存在 data_collectors.operator_id = account.code 403 collector is not registered in keystone
Keystone 本地状态 data_collectors.status == active 403 collector is inactive
这些准入规则写死在本版本中，不做环境变量配置。
06

本地数据处理

Keystone 不自动创建本地 collector。Hilbert 登录成功但本地没有绑定时拒绝登录，避免产生没有组织、工位和业务归属的半成品身份。

Hilbert displayName 非空时覆盖本地姓名，并同步工位冗余字段，避免列表、看板和任务页面展示不一致。

UPDATE data_collectors
SET name = ?, last_login_at = NOW()
WHERE id = ? AND deleted_at IS NULL;

UPDATE workstations
SET collector_name = ?, updated_at = NOW()
WHERE data_collector_id = ?
  AND deleted_at IS NULL;
07

密码字段废弃

创建数采员

POST /data_collectors 继续接受请求结构，但忽略 password 字段，新建记录写入 password_hash = NULL。

编辑数采员

PUT /data_collectors/:id 忽略 password 字段，不再更新 password_hash。

新环境不保留旧本地密码数据。登录路径永远不读取本地 password_hash。
08

配置要求

新环境强制配置 Hilbert base URL 和超时时间。base URL 为空时，数采员登录返回服务配置错误；base URL 非空时，数采员只走 Hilbert，Hilbert 调用失败不回退本地密码。

KEYSTONE_HILBERT_BASE_URL=http://192.168.120.226:8080/hilbert-be-backend
KEYSTONE_HILBERT_TIMEOUT_SECONDS=5
09

错误语义

场景 Keystone HTTP 响应语义
请求体缺字段 400 invalid request body
Hilbert 凭据错误、账号禁用、类型不符 401 invalid credentials
Hilbert 登录成功但 Keystone 未绑定 collector 403 collector is not registered in keystone
Keystone collector 非 active 403 collector is inactive
Hilbert 超时、不可达、5xx、响应格式异常 503 authentication service unavailable
Keystone 数据库或 JWT 生成失败 500 internal error
10

安全边界

Keystone 不打印密码、cipherDigest、randomKey 或 Hilbert sessionKey。Hilbert 角色和状态拒绝原因可以进入日志，但对前端统一返回认证失败。

本版本不保存 Hilbert session，因此 logout 不通知 Hilbert。下一版本如要做 Hilbert logout 和 Keystone JWT 可撤销，需要单独设计会话表和敏感 token 加密存储。

本版本接受 Keystone JWT 在过期前仍按现有无状态方式生效；这是既有 logout 行为，不在本次扩大。
11

实施计划

  1. 在 Keystone auth config 中增加 Hilbert base URL 和 timeout。
  2. 在 internal/auth 新增 Hilbert client，封装 nonce、SHA-256、AES-GCM、login 和错误分类。
  3. 修改 handlers.AuthHandler.Login，在 Hilbert 启用时替换 collector 密码校验路径。
  4. 修改 collector create/update，忽略 password 并停止写入新 password hash。
  5. 登录成功时覆盖 data_collectors.name 和 workstations.collector_name，并更新 last_login_at。
  6. 更新源码注释，不运行 Swagger 生成。
12

验证标准

自动化测试
  • Hilbert 成功 + 本地 active collector 存在，返回 Keystone JWT。
  • Hilbert 成功 + 本地 collector 不存在，返回 403。
  • Hilbert 业务认证失败，返回 401。
  • Hilbert 5xx、超时或坏 JSON，返回 503。
手工验收
  • 使用 Hilbert 账号 dc01 和正确密码可完成 Keystone 登录。
  • 错误密码不会回退 Keystone 本地 hash。
  • Hilbert displayName 非空时，本地 collector 和工位展示名被覆盖。
  • Synapse 不改代码也能继续提交原登录请求。

最终锁定

本版本是一条窄切换：Hilbert 成为数采员凭据源，Keystone 保持业务身份和 JWT 发行者。同步只处理必要的姓名一致性，不引入会话持久化和可撤销 token。
