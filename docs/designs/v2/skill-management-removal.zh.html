核心判断

本设计不是隐藏技能菜单，而是从 V2 产品模型中删除 skill 领域概念。SOP 不再描述步骤序列，只作为任务与生产流程的工艺标识存在。

01

背景与问题

当前 Keystone 仍包含 /skills CRUD、skills 表、subscene_skills 表、sops.skill_sequence 字段和 operations.skill_id。Synapse 也有技能管理页面，并在 SOP 表单中通过技能 API 维护技能序列。

代码检查确认 operations 表没有运行时读写，只存在于 schema。Axon recorder 对任务配置中的 skills 字段是可选兼容：字段缺失时保持空数组，metadata 仅在非空时写入。因此 Keystone 可以删除任务配置响应里的 skills 字段，而无需同步修改 Axon。

必须整体删除：API、schema、SOP 契约、任务配置、前端页面和 OpenAPI 不能留下半截技能模型。
02

目标与非目标

目标

  • 删除 Keystone /skills API 和 SkillHandler。
  • V2 schema 删除 skills、subscene_skills、operations。
  • SOP 请求、响应和表结构删除 skill_sequence。
  • 任务配置响应删除 skills，不再查询 skills 表。
  • Synapse 删除技能管理页面，SOP 表单只保留 slug、version、description。

非目标

  • 不新增 SOP 步骤、任务模板或操作模板。
  • 不把 skill 改名成别的新领域概念。
  • 不修改 Axon recorder 的 optional skills 兼容字段。
  • 不处理旧数据库自动迁移；V2 继续作为新基线。
  • 不清理旧 migrations/ 历史目录和历史设计文档。
03

方案总览

V2 的生产建模链路从“Subscene 关联 Skill，SOP 保存 Skill 序列，TaskConfig 输出 Skill tags”收缩为“Task 关联 SOP，SOP 作为工艺标识，TaskConfig 只输出录制所需上下文”。

05

后端修改面

删除 Skill API 删除 internal/api/handlers/skill.go，从 server.go 移除 handler 字段、初始化与路由注册。
简化 SOP CreateSOPRequest、UpdateSOPRequest、SOPResponse 和查询逻辑删除 skill_sequence。
简化 TaskConfig TaskConfig 删除 Skills，GetTaskConfig 不再读取 sops.skill_sequence 或查询 skills。

组织、场景、子场景、SOP、任务、批次和订单的主链路继续保留。删除 skill 不改变任务分配、录制、上传、QA、同步流程。

06

前端修改面

Synapse 删除技能管理路由、菜单、API wrapper 和页面。SOP 列表、详情、表单不再拉取 skills，不再展示技能序列，不再维护 slug/version 组合选择器。

页面 保留 删除
SOP 列表 / 表单 slug、version、description 技能选择器、技能版本选择、skill_sequence 校验
SOP 详情 SOP 基础信息 技能序列展示和 skills API 查询
技能管理 无 /admin/skills 页面、路由、菜单、API wrapper
07

权衡与风险

风险 影响 缓解措施
外部客户端仍调用 /skills 接口删除后旧客户端失败。 OpenAPI 删除路径，并在变更说明中标记为 breaking change。
旧 SOP 数据依赖 skill_sequence V2 新基线不自动迁移旧库，旧数据不会直接兼容。 继续明确 V2 是新基线；旧数据迁移另行脚本化。
Axon metadata 不再出现 skills 下游如果用 skills 标签检索历史数据，新数据会缺少该标签。 这是产品模型删除的直接结果；Axon 端已验证兼容字段缺失。
SOP 语义变窄 SOP 只剩工艺标识，不表达执行步骤。 未来如果需要步骤模型，重新设计 task template / procedure steps，不复用旧 skill 概念。
08

实施与验证

  1. 修改 V2 schema：删除 skills、subscene_skills、operations，从 sops 删除 skill_sequence。
  2. 删除 Keystone SkillHandler、server 注册和相关 tests；更新 Swagger 注释后重新生成 OpenAPI。
  3. 简化 SOP handler、请求响应类型、SQL 查询、测试 fixture 和校验。
  4. 简化 TaskConfig：删除 Skills 字段和 skills 表查询；更新任务配置测试。
  5. 删除 Synapse 技能 API、页面、路由、菜单；简化 SOP 列表/详情/表单。
  6. 运行 Keystone 全量测试、Synapse build、残留扫描和 diff 检查。

最终状态：SOP 是标识，不是技能序列；任务配置只携带录制上下文，不再输出技能标签。
